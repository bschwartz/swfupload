SWFUpload Revision 7 Documentation

SWFUpload

SWFUpload is a tool/widget/control developed by Mammon.se. It uses a combination Flash and JavaScript to provide file upload functionality beyond what the basic browser provides with <input type="file" />

The main features that SWFUpload are:

  • Select multiple files in the file browser dialog.
  • AJAX-style upload without a page refresh
  • Upload progress indicator
  • Graceful degradation for unsupported browsers.
  • Namespaced classes compatible with other JavaScript libraries (i.e., jQuery, Prototype, etc.)

The philosophy behind the design of SWFUpload is what makes it different from other Flash based upload tools. SWFUpload leaves developers in control by leaving the UI in the browser. Developers can use HTML, CSS, and JavaScript to tailer the upload UI to the needs of their site. Upload status updates are made through a set of simple set of JavaScript events that the developer overrides in order to update the webpage as need for their site.

Revision 7

Revision 7 is an extended version of SWFUpload that adds a number of features not found in the official SWFUpload release. These include:

  • Ability to send additional POST values with the upload.
  • Send per file POST values.
  • More refined JavaScript events.
  • Ability to update any SWFUpload settings dynamically.
  • Retrieve result data from the server.
  • Work-arounds for bugs in Flash not addressed in the official version.
  • Stop an upload without cancelling.
  • Upload files out of order.
  • Multi- & Single-file selection dialogs.
  • Bug fixes for issues discussed in the SWFUpload forums:
    • Flash fails to load on reload in I.E.
    • Flash fails to load on reload in Firefox unless the windows is scrolled back to the top.
    • Other misc race-conditions
  • Prevents user from queueing more files than they are allowed to upload.
  • Properly handles zero-byte files.
  • uploadStart event allows JavaScript validation before the upload begins.

Revision 7 is a complete re-write of the SWFUpload code. At Revision 4 code was submitted back to the SWFUpload developers which resulted in SWFUpload v1.0.2. The "Revisions" have under semi-active development while SWFUpload development has slowed/stopped.

Revision 7 continues the SWFUpload design goal of leaving the UI in the hands of developers and pushes it even further. While SWFUpload provides a default style-able UI Revision 7 provides no UI features.

Overview

HTML Upload

The standard HTML upload input box provides a box and a button to the user for selecting a file. The file is submitted with the form. The entire file must be uploaded before the next page is displayed. No pre-upload validation can be made on the file (e.g., file size limits or valid extensions).

The usage pattern for standard HTML uploads is simple, straight forward, and supported by nearly all browser.

SWFUpload

SWFUpoad uses a hidden Flash movie to handle file selection and upload. JavaScript is used to activate the file selection dialog. The file selection dialog is configured to allow the user select only one file or multiple files. The file types can be restricted so users only select the appropriate files (e.g., *.jpg;*.gif).

Once files are selected each is processed to check for valid file extensions and that the file is within size limits. Flash calls several pre-defined JavaScript functions which the developer overrides in order to update the page UI so users can see file progress or errors.

The uploaded file is submitted separately from the rest of the page and form. Each file is uploaded separately which keeps the server-side upload handling script simple as it only ever has to handle a single uploaded file at a time. Since Flash is providing the upload service the page does not have to be reloaded (behaving very much like AJAX). The usage pattern for SWFUpload is more like that of an AJAX application than that of standard HTML form. Often the server-side script that receives the uploaded file is different than the target of the page's form. The page's form will be processed separately from the file upload.
